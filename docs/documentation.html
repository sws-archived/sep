Redesign Documentation

This page documents technology and details related to the SEP website redesign project.

Style Guide

Style guide information can be found here. The style guide includes typography and color specifications, as well as style examples for various re-usable design elements, such as boxes, links, buttons, etc. The style guide also provides additional spot color ideas if you need a wider color palette.

Responsive Behavior

This site features very customized responsive behaviors for the header, navigation, footer, and article/sidebar containers. The responsive behaviors of this site are hand-coded, so if you add/remove/move/adjust any containers/code, please make sure to test at each breakpoint (you can simply re-size your browser, or use a tool like http://responsive.is).

Javascript Fallback

The templates for the site rely on javascript to function fully (with dropdown and sticky behaviors). On page load, a script runs to remove the "nojs" class from the body, as well as a couple of other classes in the menu. If a browser does not have javascript enabled, the "nojs" class will remain, and an alternate display will occur, allowing the user to access all the menus and drop-downs.

Separation of Content from Style

Just a note to be careful to separate all style from the HTML content. Ideally, never place inline styles (e.g. style="blah") in your HTML. Instead, derive semantic class names that are re-usable to you going forward, and define those styles in the stylesheet.
